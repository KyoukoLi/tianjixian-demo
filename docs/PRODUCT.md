# 天际线 — AI 互动叙事引擎产品方案

> 版本：v0.1 | 作者：@朕玥 | 日期：2026-03-23
> 状态：产品构思阶段，待评审

---

## 一、产品概念

### 1.1 一句话定位

**一个基于大语言模型的动态叙事引擎——将"读过的故事"变成"正在发生的故事"。**

用户不是选择剧本分支，而是在与 AI 角色的对话中实时推进剧情。每一个用户的"现在"都是独一无二的。

### 1.2 与现有产品的区隔

| 维度 | 传统互动小说 | 角色扮演聊天 | **天际线** |
|------|------------|------------|-----------|
| 内容生成 | 预设分支剧本 | 自由对话 | **框架约束 + AI 实时生成** |
| 沉浸感 | 文字 + 图片 | 对话为主 | **文字 + 动作 + 语音 + 形象多模态** |
| 防破戏 | 无 | 无 | **人设锁死 + 违规熔断** |
| 内容供给 | PGC 编剧 | UGC 无框架 | **模组化：PGC 模组 × UGC 共创** |

### 1.3 核心用户价值

- **情感寄托**：用户与角色建立真实的情感连接（类乙女游戏）
- **探索欲望**：每个选择通向未知，不可预测（类 TRPG）
- **创作满足**：用户参与故事推进，而非被动消费

## 二、故事框架设计

### 2.1 框架四要素

每个故事模组由四个要素定义：

```
故事模组 = 题材 + 背景 + 人物 + 框架规则
```

**题材**：故事的类型（校园/职场/悬疑/奇幻...）
**背景**：世界观设定、时间线、空间范围
**人物**：核心角色人设库（性格、目标、禁忌、语言风格）
**框架规则**：剧情推进逻辑、允许的分支范围、结局类型

### 2.2 叙事推进机制

**核心公式：**

```
当前情境 + 用户行为 → 模拟器计算 → 生成下一个情境
```

**三层推进逻辑：**

| 层级 | 触发条件 | 系统行为 |
|------|---------|---------|
| **日常对话层** | 用户闲聊、无明确目的 | 角色维持人设，积累好感/情感资本 |
| **剧情事件层** | 用户做出关键选择 | 触发预设事件节点，剧情推进 |
| **高潮冲突层** | 剧情张力达到阈值 | 强制触发核心冲突，导向阶段性结局 |

### 2.3 框架约束（"合理"）

**硬约束（绝对不能违反）：**
- 角色的核心人设（性格、价值观、语言风格）
- 故事背景的世界观规则
- 时间线逻辑（不能穿越、不产生悖论）

**软约束（尽量保持）：**
- 当前剧情的情绪基调
- 角色之间的关系状态
- 用户的情感积累资本

## 三、角色人设体系

### 3.1 角色原子数据

每个角色由以下原子数据定义：

```json
{
  "id": "char_001",
  "name": "顾深",
  "role": "学长 / 公司前辈",
  "persona": {
    "core_traits": ["外冷内热", "责任感强", "不善表达"],
    "speaking_style": ["简洁", "偶尔毒舌", "关键时候温柔"],
    "forbidden_topics": ["前任", "家庭创伤"],  // 角色不能主动提及
    "emotional_triggers": {    // 情绪触发词
      "happy": ["被认可", "轻松氛围"],
      "sad": ["无力感", "被误解"],
      "angry": ["被欺骗", "原则被触碰"]
    }
  },
  "goals": {              // 角色目标（影响剧情走向）
    "primary": "保护用户角色",
    "hidden": "解决自己的心结"
  },
  "dialogue_examples": [   // 风格示例（few-shot 给模型参考）
    "被夸奖时：*轻咳一声* ...还行吧。",
    "表达关心时：*把外套披在你肩上* ...别感冒。"
  ]
}
```

### 3.2 防破戏机制

**核心原则：AI 的第一优先不是"正确回答"，而是"不破坏沉浸感"。**

**三层防破戏体系：**

```
第一层：Prompt 硬锁
├── 系统提示词包含角色人设（不可被用户覆盖）
├── 明确禁止行为：承认自己是 AI、跳出角色、与角色人设矛盾
└── 违禁词库：自动过滤"程序"、"代码"、"我不存在"等

第二层：实时输出审核
├── 每轮 AI 输出经过轻量审核模型
├── 检测到破戏内容 → 强制重写或跳过该输出
└── 审核维度：人设一致性、剧情逻辑、情感基调

第三层：熔断机制
├── 连续 N 次输出异常 → 触发熔断
├── 熔断行为：切换到安全回复 + 记录日志
└── 熔断恢复：5 分钟后自动重试
```

**示例场景：**

```
用户：你是 AI 吗？
❌ 错误：我是大语言模型...
✅ 正确（角色顾深）：*眉头微皱* 你问这个干嘛？
```

### 3.3 情绪感知与响应

**情绪系统是沉浸感的关键锚点。**

```
用户输入 → 情绪识别 → 情绪映射 → 角色反应
     ↓
  [动作语法]
  [文本情绪]
  [历史情绪]
     ↓
  neutral / happy / sad / angry / tired
     ↓
  视频素材切换（表情匹配）
  语音语调调整（TTS 参数）
  动作动画触发
  剧情影响因子 ±
```

**情绪 → 多模态响应对照：**

| 情绪 | 视频表情 | 语音语调 | 动作动画 |
|------|---------|---------|---------|
| happy | 微笑 / 眯眼 | 轻快上扬 | 点头 / 拍手 |
| sad | 垂眼 / 垂眉 | 低沉放缓 | 叹气 / 垂手 |
| angry | 皱眉 / 抿嘴 | 压抑下沉 | 握拳 / 转身 |
| neutral | 平静 | 自然平稳 | 自然站立 |
| tired | 眼皮下垂 | 有气无力 | 揉眼 / 撑头 |

## 四、模组化内容体系

### 4.1 模组生命周期

```
模组制作 → 审核 → 上架 → 运营监控 → 迭代优化 → 下架
```

### 4.2 模组供给策略

**PGC（平台自制）：**
- 专业编剧团队打造精品核心模组
- 重点 IP 联动（快手自有达人 / 影视版权）
- 标杆效应，吸引 UGC

**PUGC（专业用户共创）：**
- 提供模组编辑器 + 角色人设模板
- 平台提供 AI 能力 + 安全审核服务
- 分成机制激励创作者

**UGC（自由创作）：**
- 基础模组模板，降低创作门槛
- AI 辅助生成角色人设和剧情节点
- 社区分享 + 评分体系

### 4.3 模组质量标准

| 等级 | 要求 | 审核方式 |
|------|------|---------|
| S | 剧情完整、有高潮、结局感人 | 人工评审 |
| A | 逻辑自洽、沉浸感强 | 人工 + AI 双审 |
| B | 人设清晰、无明显破戏 | AI 自动审核 |
| C | 基本可玩 | AI 初审 + 用户反馈 |

## 五、交互产品设计

### 5.1 产品形态

**MVP（当前 Demo 阶段）：**
```
单一角色 × 单一场景 × 纯文本交互 + 情绪渲染
```

**v1.0（首个正式版本）：**
```
多角色 × 场景切换 × 文字 + 动作 + 语音 + 形象
```

**v2.0（长期愿景）：**
```
开放世界 × UGC 模组 × 社交分享 × 多端联动
```

### 5.2 核心交互设计

**消息输入：**

| 输入类型 | 示例 | AI 理解 |
|---------|------|---------|
| 纯对话 | "你好" | 普通对话流 |
| 带动作 | "你好*微笑*" | 动作语法，触发对应情绪 |
| 带选择 | "我选择A" | 分支剧情推进 |
| 带探索 | "我观察周围环境" | 场景细节描述触发 |

**剧情推进 UI：**

- **继续按钮**（当前 Demo 已实现）：推进当前剧情节点
- **关键选择卡片**：剧情关键节点时，弹出选项卡片
- **剧情时间轴**：显示当前进度，增强叙事感

### 5.3 商业化路径

**第一阶段：免费体验**
- 每日免费对话轮次（20 轮/天）
- 体验核心剧情的"免费预览"章节

**第二阶段：内容付费**
- 单个精品模组付费（6-30 元）
- 季卡/年卡解锁全部模组

**第三阶段：生态分成**
- UGC 模组付费分成（创作者 60% / 平台 40%）
- 角色定制服务（AI 生成个性化角色）
- 品牌联动植入

## 六、技术架构

### 6.1 整体架构

```
用户端（App / Web）
    ↓ HTTPS / WebSocket
API 网关（鉴权 + 限流）
    ↓
对话服务
├── 角色引擎（Prompt 组装 + 防破戏）
├── 剧情引擎（故事框架 + 事件节点）
└── 情绪引擎（情绪识别 + 多模态渲染参数）
    ↓
LLM 层
├── 推理服务（自有 GPU 集群 / 火山引擎）
└── 降级路由（主模型 → 备选模型）
    ↓
多模态渲染服务
├── 视频素材调度（CDN）
├── TTS 语音合成
└── 动作动画触发
    ↓
存储层
├── 会话状态（Redis）
├── 用户数据（MySQL）
└── 模组内容（对象存储）
```

### 6.2 核心服务拆分

| 服务 | 职责 | 独立性 |
|------|------|--------|
| **对话服务** | Prompt 组装、流式输出 | 核心，可复用 |
| **角色引擎** | 人设加载、防破戏审核 | 强依赖人设数据 |
| **剧情引擎** | 故事节点、事件触发 | 强依赖模组数据 |
| **情绪引擎** | 情绪识别、渲染参数输出 | 中等独立 |
| **素材服务** | 视频/音频 CDN 调度 | 完全独立 |
| **审核服务** | 内容安全、破戏检测 | 可复用 |

### 6.3 当前 Demo 定位

**Demo 是对话引擎的技术验证**（已完成 ✅）

验证内容：
- 情绪识别链路跑通
- 前端多模态调度框架就绪
- 降级策略有效

**待验证（下一阶段）：**
- 角色人设 Prompt 约束有效性
- 防破戏审核机制
- 多角色场景切换

## 七、关键指标体系

### 7.1 用户体验指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| 沉浸感得分 | 用户调研（NPS 追问：为什么不沉浸？） | ≥ 4.0/5 |
| 单次使用时长 | 从进入模组到主动退出的时间 | ≥ 8 分钟 |
| 继续率 | 点击"继续"按钮的次数 / 对话轮次 | ≥ 30% |
| 破戏率 | 触发破戏审核的次数 / 总对话轮次 | ≤ 2% |

### 7.2 产品健康指标

| 指标 | 定义 | 预警阈值 |
|------|------|---------|
| TTFB | 首 token 响应时间 | > 2s 告警 |
| 渲染成功率 | 视频/音频正常加载占比 | < 95% 告警 |
| 模型 SLA | LLM 服务可用率 | < 99% 告警 |
| 用户留存 D7 | 7 日回访率 | < 15% 预警 |

## 八、风险与应对

| 风险 | 等级 | 应对策略 |
|------|------|---------|
| AI 生成内容违规 | 🔴 高 | 三层防破戏体系 + 人工抽审 |
| 沉浸感不足 | 🔴 高 | 早期用户共创、快速迭代 |
| 内容供给不足 | 🟡 中 | PGC 先行，PUGC 激励，UGC 开放 |
| 算力成本过高 | 🟡 中 | 模型降级路由 + 缓存复用 |
| 竞品复制 | 🟡 中 | 数据飞轮 + 模组生态壁垒 |
